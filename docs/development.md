# Development

## Helm Chart

### Requirements

If you want to develop the helm chart, you should probably have a local cluster running with e.g [kind](https://kind.sigs.k8s.io/) and [cloud-provider-for-KIND](https://github.com/kubernetes-sigs/cloud-provider-kind) (handles e.g ingress). Besides that, you also should have `kubectl` instaleld.

The following helm chart values work well with the [kind](https://kind.sigs.k8s.io/) + [cloud-provider-for-KIND](https://github.com/kubernetes-sigs/cloud-provider-kind) setup.

```yaml
# kind.yaml
apiService:
  ingress:
    enabled: true
    className: cloud-provider-kind
    hosts:
      - host: guggr.internal
        paths:
          - path: /api
            pathType: Prefix

frontend:
  ingress:
    enabled: true
    className: cloud-provider-kind
    hosts:
      - host: guggr.internal
        paths:
          - path: /
            pathType: Prefix
```

You can deploy it by running:

```sh
helm install guggr -f kind.yaml charts/guggr
```

You need to run [cloud-provider-for-KIND](https://github.com/kubernetes-sigs/cloud-provider-kind) in the background or in another terminal window with elevated permissions so guggr can be reachable locally.
In order for you to be able to reach guggr, you need to add the IP that is assigned to the ingress to your `/etc/hosts`. This IP can be retrieved by running:

```sh
kubectl get ingress
# example output:
NAME                CLASS                 HOSTS            ADDRESS                            PORTS   AGE
guggr-api-service   cloud-provider-kind   guggr.internal   172.22.0.3,fc00:f853:ccd:e793::3   80      4d15h
guggr-frontend      cloud-provider-kind   guggr.internal   172.22.0.3,fc00:f853:ccd:e793::3   80      4d15h
```

Then you would need to add the following row (at the top) to the `/etc/hosts` file:

```
172.22.0.3 guggr.internal
```

Now you should be able to reach guggr via http://guggr.internal

### Development notes

You need to run the following command to build the charts' dependencies (required once):

```sh
helm dependency build charts/guggr
```

The `README.md` inside `charts/guggr` is partly autogenerated. All Subsections under `## Parameters` are generated from the `values.yaml` with:

```sh
# (also run via our pre-commit hooks)
pnpm readme-generator -v charts/guggr/values.yaml -r charts/guggr/README.md
```

So do **not** change content in the `## Parameters` section. If you need to change something in that sections, go to the equivalent sections inside the `values.yaml`.
